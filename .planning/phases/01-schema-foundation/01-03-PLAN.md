  <action>
Create `src/observatory/ddl.py` with:

- `DDLDialect` enum (or string literal type) with values: "postgres", "bigquery"

- `TYPE_MAP: dict` mapping dialect to column type:
  - "postgres" -> "TEXT"
  - "bigquery" -> "STRING"

Since ALL columns are TEXT/STRING in the ELT approach, there is no per-column type logic. Every column gets the same type. This is intentional — the user explicitly decided on ELT where SQL queries handle type casting.

- `generate_create_table(schema: ResourceSchema, dialect: str) -> str`:
  1. If schema.is_empty (no columns discovered):
     - Return a commented-out placeholder:
       ```sql
       -- Table: {table_name}
       -- No records found in sample data. Columns unknown.
       -- Add columns manually when data becomes available.
       -- CREATE TABLE {table_name} ();
       ```
  2. For non-empty schemas:
     - Generate CREATE TABLE IF NOT EXISTS {table_name} (
     - One line per column: `  {column_name} {TYPE}` (TEXT for postgres, STRING for bigquery)
     - Columns separated by commas
     - Closing );
     - Add a header comment with resource type name and record count from sample
  3. For BigQuery dialect:
     - Use backtick quoting for table names if they contain special characters (they won't for these names, but be safe)
     - No IF NOT EXISTS (BigQuery uses different syntax) — use CREATE TABLE with no qualifier, add a comment noting "-- Use CREATE TABLE IF NOT EXISTS for BigQuery by wrapping in a script or using DDL options"

     Actually, BigQuery DOES support CREATE TABLE IF NOT EXISTS. Use it for both dialects.
  4. Column names: use the exact snake_case column names from the Particle data (they're already snake_case). Do NOT quote column names unless they conflict with SQL reserved words. For safety, quote ALL column names with double quotes (postgres) or backticks (bigquery) since some column names like "text", "type", "status", "name" ARE reserved words in SQL.

- `generate_ddl(schemas: list[ResourceSchema], dialect: str) -> str`:
  1. Generate a file header comment:
     ```sql
     -- Particle Flat Data Observatory
     -- DDL for {dialect} — Generated from sample data
     -- All columns are {TYPE} (ELT approach: transform in queries, not on load)
     --
     -- Resource types: {count} total, {non_empty} with data, {empty} empty
     -- Generated: {ISO timestamp}
     ```
  2. For each ResourceSchema in order, call generate_create_table and join with blank lines
  3. Return the complete SQL string

- `write_ddl(sql: str, output_path: str | Path) -> None`:
  1. Create parent directories if they don't exist (pathlib.Path.mkdir(parents=True, exist_ok=True))
  2. Write the SQL string to the file
  3. Log: "DDL written to {output_path}"

Create `tests/test_ddl.py` with tests:
- test_generate_create_table_postgres: single ResourceSchema with 3 columns, verify correct CREATE TABLE with TEXT type and double-quoted column names
- test_generate_create_table_bigquery: same schema, verify STRING type and backtick-quoted column names
- test_generate_create_table_empty: empty ResourceSchema, verify commented-out placeholder
- test_generate_ddl_multiple_tables: list of schemas, verify all tables present in output
- test_reserved_word_columns: ResourceSchema with columns named "text", "type", "status" — verify they're properly quoted
- test_column_order_preserved: verify output column order matches input ResourceSchema.columns order
- test_generate_ddl_header: verify header comment contains dialect name and resource count

Update `src/observatory/__init__.py` to export generate_ddl, DDLDialect, write_ddl.

Create `src/observatory/generate_ddl.py` as a runnable script (not just a module):

- Uses argparse (stdlib) for CLI:
  - `--data-path`: Path to flat_data.json (default from config/env: FLAT_DATA_PATH)
  - `--dialect`: "postgres", "bigquery", or "all" (default: "all")
  - `--output-dir`: Output directory (default from config/env: OUTPUT_DIR, which defaults to "ddl")
  - `--no-normalize`: Skip empty string -> NULL normalization (for debugging)

- Main flow:
  1. Parse CLI args, merge with environment config (CLI args override env vars)
  2. Configure logging (stdlib logging, format: "%(levelname)s: %(message)s")
  3. Load flat data via `load_flat_data(data_path, normalize=not args.no_normalize)`
  4. Inspect schema via `inspect_schema(data)`
  5. For each requested dialect:
     a. Generate DDL via `generate_ddl(schemas, dialect)`
     b. Write to `{output_dir}/{dialect}/create_all.sql` via `write_ddl(sql, path)`
  6. Print summary to stdout:
     ```
     Generated DDL for {dialect}:
       Tables: {total} ({non_empty} with columns, {empty} empty)
       Output: {output_path}
     ```

- Add `if __name__ == "__main__":` block so it can be run as `python -m observatory.generate_ddl`

Also register it as a console_scripts entry point in pyproject.toml:
```toml
[project.scripts]
observatory-generate-ddl = "observatory.generate_ddl:main"
```

After creating the script, RUN IT to generate the actual DDL files:
