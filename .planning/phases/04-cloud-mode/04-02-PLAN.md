**bq_loader.py** -- Mirror the structure of loader.py. Create three functions:

1. `get_bq_client()`:
   - Read BQ_PROJECT_ID from os.environ. If not set, raise ValueError with actionable message: "BQ_PROJECT_ID not set. Set BQ_PROJECT_ID in .env or environment to your GCP project ID."
   - Read BQ_DATASET from os.environ with default "particle_observatory".
   - Return tuple of (bigquery.Client(project=project_id), dataset_id).
   - Use `from google.cloud import bigquery` at module top.

2. `load_resource_bq(client, dataset_id, table_name, columns, records, patient_id)`:
   - If not records, return 0 (same skip pattern as loader.py).
   - Build table_ref as f"`{client.project}.{dataset_id}.{table_name}`" for the DELETE query.
   - Step 1 (DELETE): Build parameterized DELETE query: `DELETE FROM {table_ref} WHERE \`patient_id\` = @patient_id`. Use bigquery.QueryJobConfig with ScalarQueryParameter("patient_id", "STRING", patient_id). Call client.query(query, job_config=config).result() to wait for completion.
   - Step 2 (INSERT): Build schema as [bigquery.SchemaField(col, "STRING", mode="NULLABLE") for col in columns]. Build rows as [{col: record.get(col) for col in columns} for record in records]. Use bigquery.LoadJobConfig(schema=schema). Call client.load_table_from_json(rows, f"{client.project}.{dataset_id}.{table_name}", job_config=load_config).result().
   - Log records loaded using logger.info (same pattern as loader.py).
   - Return len(records).
   - IMPORTANT: Use backtick-quoted table ref for the DELETE query string (BigQuery syntax), but use the unquoted dotted path for load_table_from_json (which takes a table reference, not SQL).

3. `load_all_bq(client, dataset_id, data, schemas)`:
   - Same structure as load_all in loader.py: iterate schemas, skip empty, group by patient_id, call load_resource_bq per patient.
   - Return dict[str, int] mapping table_name -> total records loaded.
   - Log completion summary.

8. `test_load_resource_bq_uses_parameterized_delete` -- Verifies QueryJobConfig has ScalarQueryParameter with patient_id.
9. `test_load_all_bq_skips_empty_schemas` -- Empty schemas are skipped.
10. `test_load_all_bq_groups_by_patient` -- Multiple patients' records are loaded separately.

Use `unittest.mock.patch` to mock `google.cloud.bigquery` at the `observatory.bq_loader` module level. Use `@patch.dict(os.environ, ...)` for env var tests.

IMPORTANT: Since google-cloud-bigquery may not be installed in the test environment, mock the entire google.cloud.bigquery module at import time. Use `unittest.mock.MagicMock` for the bigquery module and patch it into sys.modules if needed, OR structure tests to mock at the function level.

Simpler approach: Mock at function call boundaries. In test setup, use `@patch("observatory.bq_loader.bigquery")` to replace the bigquery module reference. If the google-cloud-bigquery package is not installed, the import at module top will fail. To handle this, use a try/except in bq_loader.py:

Then in get_bq_client(), check: if bigquery is None, raise ImportError with message "google-cloud-bigquery is not installed. Install with: pip install -e '.[bigquery]'".
