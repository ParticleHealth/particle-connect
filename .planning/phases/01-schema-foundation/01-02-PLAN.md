  <action>
Create `src/observatory/normalizer.py` with:

- `normalize_value(value: Any) -> Any`: If value is an empty string "", return None. Otherwise return value unchanged. This is the ONLY transformation in the ELT pipeline.
- `normalize_record(record: dict) -> dict`: Apply normalize_value to every value in the dict. Return a new dict (do not mutate input). Preserve key order.
- `normalize_resource(records: list[dict]) -> list[dict]`: Apply normalize_record to every record in the list. Return a new list.

Keep it dead simple. The ELT philosophy means we store raw data and transform in SQL. The only load-time transformation is empty string -> NULL because Particle returns "" instead of null for missing values.

Create `src/observatory/parser.py` with:

- `EXPECTED_RESOURCE_TYPES: list[str]` — hardcoded list of all 21 resource type keys in the order they appear in the Particle API response: aICitations, aIOutputs, allergies, coverages, documentReferences, encounters, familyMemberHistories, immunizations, labs, locations, medications, organizations, patients, practitioners, problems, procedures, recordSources, socialHistories, sources, transitions, vitalSigns
- `load_flat_data(path: str | Path, normalize: bool = True) -> dict[str, list[dict]]`:
  1. Read and parse JSON file at path
  2. Validate top-level structure is a dict
  3. For each key in the parsed dict:
     - If key is in EXPECTED_RESOURCE_TYPES: include it
     - If key is NOT in EXPECTED_RESOURCE_TYPES: log a warning ("Unknown resource type: {key}, skipping") and skip it
  4. For each expected resource type NOT in the parsed dict: log info ("Resource type {key} not found in data, treating as empty") and include it with empty list
  5. If normalize=True, apply normalize_resource to each resource's records
  6. Return dict with all 21 resource type keys (some may have empty lists)
  7. Raise FileNotFoundError with actionable message if file not found
  8. Raise json.JSONDecodeError (let it propagate) if invalid JSON
  9. Raise ValueError if top-level structure is not a dict

Use Python stdlib logging module: `logger = logging.getLogger(__name__)`
Log a summary after loading: "Loaded {N} resource types: {types_with_counts}" where types_with_counts shows each type and its record count.
Log at INFO level for normal operations, WARNING for unexpected data.

Update `src/observatory/__init__.py` to export load_flat_data, normalize_record, normalize_resource from these modules.

Create `src/observatory/schema.py` with:

- `ResourceSchema` dataclass (or simple class) with fields:
  - `resource_type: str` — the camelCase key (e.g., "aICitations")
  - `table_name: str` — snake_case version for SQL (e.g., "ai_citations")
  - `columns: list[str]` — ordered list of column names discovered from data
  - `record_count: int` — number of records found
  - `is_empty: bool` — True if no records (0 records)

- `camel_to_snake(name: str) -> str`: Convert camelCase resource type keys to snake_case table names. Handle edge cases:
  - "aICitations" -> "ai_citations"
  - "aIOutputs" -> "ai_outputs"
  - "documentReferences" -> "document_references"
  - "vitalSigns" -> "vital_signs"
  - "recordSources" -> "record_sources"
  - "socialHistories" -> "social_histories"
  - "familyMemberHistories" -> "family_member_histories"
  Use a regex approach: insert underscore before uppercase letters, then lowercase. Special-case the "aI" prefix to produce "ai_" not "a_i_".

- `inspect_schema(data: dict[str, list[dict]]) -> list[ResourceSchema]`:
  1. For each resource type in data:
     - Scan ALL records (not just the first) to discover the full set of column names. Some records may have fields that others don't.
     - Preserve the order columns are first seen (use dict.fromkeys() or an OrderedDict approach to deduplicate while preserving insertion order).
     - Create ResourceSchema with the discovered columns.
  2. For empty resource types (0 records): columns will be an empty list. This is expected — the DDL generator will handle it.
  3. Return list of ResourceSchema objects sorted by the order in EXPECTED_RESOURCE_TYPES.
  4. Log summary: "{N} resource types inspected, {M} non-empty, {K} empty"
