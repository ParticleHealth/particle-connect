**data_completeness.sql (OPS-01):**
- Data completeness scorecard across all resource types with data (16 tables).
- Use UNION ALL to query each table separately. For each table, report:
  - resource_type (hardcoded string)
  - record_count (COUNT(*))
  - key_field_populated (COUNT of the most important non-null field for that table)
  - key_field_pct (ROUND(100.0 * key_field_populated / NULLIF(record_count, 0), 1))
- Key fields per table:
  - patients: "given_name"
  - encounters: "encounter_type_name"
  - problems: "condition_name"
  - medications: "medication_name"
  - labs: "lab_name"
  - vital_signs: "vital_sign_observation_name"
  - procedures: "procedure_name"
  - practitioners: "practitioner_given_name"
  - organizations: "organization_name"
  - locations: "location_name"
  - document_references: "document_reference_type"
  - sources: "source_name"
  - record_sources: "resource_id"
  - ai_outputs: "type"
  - ai_citations: "resource_reference_id"
  - transitions: "facility_name"
- ORDER BY record_count DESC

**source_coverage.sql (OPS-02):**
- Join record_sources to sources on source_id to get source_name.
- GROUP BY source_name, resource_type.
- Show: source_name, resource_type, record_count, percentage of total for that resource_type.
- Use a CTE for totals per resource_type, then join for percentage calculation.
- ORDER BY source_name, record_count DESC.

**record_freshness.sql (OPS-03):**
- For each resource type that has a timestamp column, show the most recent record.
- Use UNION ALL with one query per table:
  - encounters: MAX(CAST("encounter_start_time" AS TIMESTAMPTZ))
  - problems: MAX(CAST("condition_onset_date" AS TIMESTAMPTZ))
  - medications: MAX(CAST("medication_statement_start_time" AS TIMESTAMPTZ))
  - labs: MAX(CAST("lab_timestamp" AS TIMESTAMPTZ))
  - vital_signs: MAX(CAST(LTRIM("vital_sign_observation_time", ', ') AS TIMESTAMPTZ)) -- leading comma!
  - procedures: MAX(CAST("procedure_date_time" AS TIMESTAMPTZ))
  - ai_outputs: MAX(CAST("created" AS TIMESTAMPTZ))
  - transitions: MAX(CAST("status_date_time" AS TIMESTAMPTZ))
- For tables without timestamps (patients, practitioners, organizations, locations, document_references, sources, record_sources, ai_citations): show NULL for most_recent.
- Include record_count alongside most_recent.
- ORDER BY most_recent DESC NULLS LAST.

- Join record_sources to sources on source_id.

  - Second query (below a `-- Detail: full provenance trace` comment): Full detail showing resource_type, resource_id, resource_id_name, source_name, ORDER BY resource_type, resource_id

Create the directory structure `particle-flat-observatory/queries/bigquery/operational/` and write 5 SQL files.

These are BigQuery dialect equivalents of the PostgreSQL operational queries from Task 1. Same logic, different syntax.

Key dialect transformations to apply:

1. **Identifier quoting:** Backticks instead of double quotes
2. **Timestamp parsing:**
   - For +0000 offsets: `PARSE_TIMESTAMP('%Y-%m-%dT%H:%M:%S%z', col)`
   - For no-timezone ISO: `SAFE_CAST(col AS TIMESTAMP)`
   - For fractional seconds + Z: `PARSE_TIMESTAMP('%Y-%m-%dT%H:%M:%E*SZ', col)`
   - For vital signs leading comma: `PARSE_TIMESTAMP('%Y-%m-%dT%H:%M:%S%z', LTRIM(col, ', '))`
   - For transitions status_date_time (microseconds, no Z): `PARSE_TIMESTAMP('%Y-%m-%dT%H:%M:%E*S', col)`
3. **ROUND:** Same syntax in both dialects
4. **LEFT function:** BigQuery uses `LEFT(col, n)` same as PostgreSQL -- this works identically
5. **COUNT/GROUP BY/UNION ALL:** Identical in both dialects
6. **Parameter placeholder:** `-- Parameterized: WHERE patient_id = @patient_id`

Apply these transformations to each of the 5 queries. The business logic, CTE structure, and column selections remain identical. Only the quoting and casting syntax changes.

**Specific notes:**
- record_freshness.sql: Use PARSE_TIMESTAMP for each table's timestamp column as specified above. The vital_signs LTRIM pattern stays the same.
- data_completeness.sql: This is pure COUNT/GROUP BY -- minimal dialect difference, just identifier quoting.
- ai_output_summary.sql: The `created` column in ai_outputs has fractional seconds + Z suffix, so use `PARSE_TIMESTAMP('%Y-%m-%dT%H:%M:%E*SZ', col)`.
