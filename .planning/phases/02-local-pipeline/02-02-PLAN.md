<task type="auto">
  <name>Task 1: Create PostgreSQL loader module</name>
  <files>particle-flat-observatory/src/observatory/loader.py</files>
  <action>
Create `src/observatory/loader.py` with these functions:

**get_connection_string() -> str:**
- Build PostgreSQL connection string from env vars: PG_HOST (default "localhost"), PG_PORT (default "5432"), PG_USER (default "observatory"), PG_PASSWORD (default "observatory"), PG_DATABASE (default "observatory")
- Return format: `postgresql://{user}:{password}@{host}:{port}/{dbname}`
- Read env vars via `os.environ.get()` -- do NOT use config.py's load_settings() since these are different config concerns

**load_resource(conn: psycopg.Connection, table_name: str, columns: list[str], records: list[dict], patient_id: str) -> int:**
- If records is empty, return 0 immediately (skip empty resource types)
- Within a single `conn.transaction()` block:
  1. DELETE all rows WHERE "patient_id" = %s for the given patient_id
  2. Build INSERT query using `psycopg.sql.Identifier` for table name and ALL column names (they must be quoted -- columns like "type", "text", "status" are SQL reserved words)
  3. Use `psycopg.sql.Placeholder()` for parameter placeholders
  4. Extract values from each record dict using the columns list: `tuple(record.get(col) for col in columns)` -- use .get() to handle records missing some columns (returns None)
  5. Use `conn.executemany(insert_query, rows)` for batch insert
- Return the number of records inserted
- Log at INFO level: table name, patient_id, records deleted (if trackable) and inserted count

**CRITICAL psycopg 3 patterns:**
- Use `from psycopg import sql` for dynamic SQL (never f-strings for SQL)
- Use `conn.transaction()` for transaction blocks (NOT conn.autocommit)
- `with psycopg.connect(...)` CLOSES the connection on exit (not commit) -- this is a breaking change from psycopg2
- Commit happens automatically when `conn.transaction()` block exits normally; rollback on exception

**load_all(conn: psycopg.Connection, data: dict[str, list[dict]], schemas: list[ResourceSchema]) -> dict[str, int]:**
- Iterate over schemas (not data keys) to maintain consistent ordering
- Skip schemas where `schema.is_empty` is True (no table exists for empty resource types)
- For each non-empty schema:
  - Get the records from `data[schema.resource_type]`
  - If no records, skip
  - Extract unique patient_ids from records (look for "patient_id" key in each record)
  - For EACH patient_id, call load_resource() with records filtered to that patient_id
  - This ensures idempotency works per-patient: delete only that patient's rows, then insert
- Return dict mapping table_name -> total records loaded
- Log summary at INFO level: total tables loaded, total records loaded

**Important design decisions:**
- Use `schema.columns` as the single source of truth for column order (matches DDL column order)
- Use `schema.table_name` for the SQL table name (already snake_case from Phase 1)
- The transaction is per load_resource call (per patient per resource type), NOT per entire load. This way a failure in one resource type does not roll back all others.
- Import logging and use `logger = logging.getLogger(__name__)`
