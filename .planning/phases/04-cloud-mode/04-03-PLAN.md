The Cloud Mode section should cover these topics in order:

**1. Prerequisites**
- Google Cloud account with a project
- Terraform >= 1.0 installed
- gcloud CLI installed
- Python 3.11+ (already required)

**2. Install BigQuery Support**
```bash
pip install -e ".[bigquery]"
```
Explain this is an optional dependency -- the base install works without it for local-only users.

**3. Authenticate**
```bash
gcloud auth application-default login
```
Explain ADC: the BigQuery client auto-discovers credentials. For CI/production, mention GOOGLE_APPLICATION_CREDENTIALS env var pointing to a service account key file.

**4. Provision Infrastructure with Terraform**
Step-by-step:
```bash
cd terraform/
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars -- set project_id to your GCP project
terraform init
terraform plan    # Review: 1 dataset, 21 tables, 1 service account, 2 IAM bindings
terraform apply
```

List the Terraform variables in a table:
| Variable | Default | Description |
|----------|---------|-------------|
| project_id | (required) | GCP project ID |
| dataset_name | particle_observatory | BigQuery dataset name |
| region | US | Dataset location |

**5. Configure Environment**
Add to .env:
```
BQ_PROJECT_ID=your-gcp-project-id
BQ_DATASET=particle_observatory
```

**6. Load Data**
```bash
particle-pipeline load --source file --target bigquery
```
Expected output: similar to local mode -- record counts per table, quality report.
Mention idempotency: safe to re-run, will delete+reinsert per patient.

**7. Run Analytics Queries**
Two options:
- **BigQuery Console:** Navigate to your project > BigQuery > select the dataset > run queries from `queries/bigquery/` directory. Note: you must select the dataset as the default in the query editor, OR prefix table names with `project.dataset.`.
- **bq CLI:**
  ```bash
  bq query --use_legacy_sql=false --dataset_id=particle_observatory < queries/bigquery/clinical/patient_summary.sql
  ```

Include a note about unqualified table names: the analytics queries use unqualified table names (e.g., `FROM patients` not `FROM project.dataset.patients`). In the BigQuery console, select the dataset first. With the bq CLI, use --dataset_id.

**8. Tear Down (optional)**
```bash
cd terraform/
terraform destroy
```
Note: tables have deletion_protection disabled for this accelerator. Production deployments should set deletion_protection = true.

**Known Limitations section (within Cloud Mode):**
- Load job quota: 1,500 per table per day. For production with many patients, batch multiple patients per load job.
- Non-atomic delete+insert: DELETE runs as DML query, INSERT runs as load job. These are not in a single transaction. Idempotent (safe to re-run) but not atomic.
- DML concurrency: BigQuery limits to 2 active + 20 queued DML operations per table. Not a concern for single-user accelerator use.

Do NOT rewrite or restructure the existing README content. Only ADD the new Cloud Mode section.
