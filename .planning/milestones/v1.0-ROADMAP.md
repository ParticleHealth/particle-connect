# Milestone v1.0: Pipeline Accelerator

**Status:** SHIPPED 2026-02-08
**Phases:** 1-5
**Total Plans:** 15

## Overview

This roadmap delivers a customer-facing data pipeline accelerator that takes Particle Health's GET Flat response data from raw JSON to queryable structured tables with pre-built analytics queries. The journey starts with schema foundation and data normalization (the hardest part due to healthcare data quality issues), delivers a working local pipeline with Docker PostgreSQL, adds a comprehensive analytics query library, extends to cloud BigQuery with Terraform, and finishes with live Particle API ingestion. Each phase delivers a complete, verifiable capability that customers can use independently.

## Milestone Summary

**Key Decisions:**

- Stdlib-only dependencies for maximum portability (no pydantic, no structlog, no httpx)
- ELT approach: all columns TEXT/STRING, transform in SQL queries downstream
- python-dotenv promoted from optional to required for CLI .env loading
- Schema inspector scans ALL records per resource type to discover full column set
- camelCase "aI" prefix special-cased to produce "ai_" not "a_i_"
- All column names quoted unconditionally to handle SQL reserved words
- Empty resource types get commented-out DDL placeholders, not empty tables
- Transaction scope per-patient per-resource-type for failure isolation
- google-cloud-bigquery as optional dependency (not required for postgres-only users)
- Terraform for_each with locals map for 21 tables -- single resource block, DRY
- Stdlib-only HTTP via urllib.request for API client -- no httpx or tenacity
- JWT exp claim decoded via base64url without signature verification

**Issues Resolved:**
- Bash wave extraction returning garbled output (fixed with awk approach)
- Docker Desktop unavailable during Phase 2 execution (deferred verification)

**Technical Debt:**
- Docker e2e execution never verified (code complete, Docker Desktop unavailable)
