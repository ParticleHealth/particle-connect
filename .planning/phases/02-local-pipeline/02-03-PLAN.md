Create `src/observatory/cli.py` with:

**Top-level setup:**
- Load .env at module level (before typer processes env vars):
  ```python
  from dotenv import load_dotenv
  load_dotenv()
  ```
- Create typer app:
  ```python
  app = typer.Typer(
      name="particle-pipeline",
      help="Particle Health flat data pipeline -- load, transform, and analyze.",
      no_args_is_help=True,
  )
  ```
- Set up logging: configure root logger with format including timestamp, level, and message. Level from LOG_LEVEL env var (default INFO).

**load command:**
```python
@app.command()
def load(
    source: Annotated[str, typer.Option("--source", help="Data source: 'file' or 'api'")] = "file",
    target: Annotated[str, typer.Option("--target", help="Target database: 'postgres' or 'bigquery'")] = "postgres",
    data_path: Annotated[str, typer.Option("--data-path", help="Path to flat_data.json", envvar="FLAT_DATA_PATH")] = "sample-data/flat_data.json",
    verbose: Annotated[bool, typer.Option("--verbose", "-v", help="Enable debug logging")] = False,
):
```

**load command body (source=file, target=postgres path):**
1. Validate source: must be "file" or "api". If "api", print message "API source not yet implemented (coming in Phase 5)" and raise typer.Exit(code=1). Use typer.echo for user messages, logger for debug.
2. Validate target: must be "postgres" or "bigquery". If "bigquery", print "BigQuery target not yet implemented (coming in Phase 4)" and raise typer.Exit(code=1).
3. If verbose, set logging level to DEBUG.
4. Print header: "Loading Particle flat data..." with source and target info.
5. Load data: call `load_flat_data(data_path)` -- wrap in try/except for FileNotFoundError with actionable message: "Data file not found: {path}\n\nTo fix: check --data-path or set FLAT_DATA_PATH in .env"
6. Inspect schema: call `inspect_schema(data)` to get ResourceSchema list
7. Print summary of data found: count of non-empty resource types, total records
8. Connect to PostgreSQL: call `psycopg.connect(get_connection_string())` -- wrap in try/except for psycopg.OperationalError with actionable message:
   - "Could not connect to PostgreSQL at {host}:{port}\n\nTo fix:\n  1. Start the database: docker compose up -d\n  2. Wait for healthy: docker compose ps\n  3. Check connection: PG_HOST={host} PG_PORT={port} in .env"
9. Load all resources: call `load_all(conn, data, schemas)`
10. Print results: count of tables loaded, total records loaded
11. Close connection when done (use `with psycopg.connect(...)` -- remember in psycopg 3 this CLOSES the connection, does NOT commit; transactions are committed via conn.transaction() blocks inside load_all)

**Error handling strategy:**
- Catch specific exceptions, NOT bare except
- Each error message follows the pattern: "What went wrong" + "To fix: [actionable steps]"
- Use `typer.echo()` for user-facing messages (not print())
- Use `raise typer.Exit(code=1)` for handled errors (not sys.exit)
- Let unexpected errors propagate with full traceback (don't swallow them)

**Do NOT:**
- Add the data quality report in this plan (that is Plan 02-04)
- Import or call anything from quality.py
- Use print() for user messages (use typer.echo)
- Use sys.exit (use typer.Exit)
- Catch bare Exception (catch specific types)
