Create the directory structure `particle-flat-observatory/queries/bigquery/clinical/` and write 7 SQL files.

These are BigQuery dialect equivalents of the PostgreSQL clinical queries from Task 1. Same logic, different syntax.

Every file uses the same header template but with `-- Dialect: BigQuery` and parameterized form `-- Parameterized: WHERE patient_id = @patient_id`.

Key dialect differences to apply consistently:

1. **Identifier quoting:** Use backticks instead of double quotes: `` `column_name` `` not `"column_name"`
2. **Timestamp parsing:** Use `PARSE_TIMESTAMP('%Y-%m-%dT%H:%M:%S%z', col)` for timestamps with timezone offset (+0000). Use `SAFE_CAST(col AS TIMESTAMP)` for simple ISO formats (date_of_birth). For vital_sign_observation_time, strip comma first: `PARSE_TIMESTAMP('%Y-%m-%dT%H:%M:%S%z', LTRIM(vital_sign_observation_time, ', '))`
3. **Numeric casting:** Use `SAFE_CAST(col AS FLOAT64)` instead of `col::NUMERIC`. SAFE_CAST returns NULL on failure (no error).
4. **String aggregation:** `STRING_AGG(expr, delim)` works the same in both, but BigQuery does not support ORDER BY inside STRING_AGG in all contexts. Use `STRING_AGG(expr, delim ORDER BY col)` -- BigQuery does support this.
5. **Array splitting:** Use `SPLIT(col, ', ')` + `UNNEST(...)` instead of `string_to_array` + `unnest`. Syntax: `UNNEST(SPLIT(col, ', ')) AS element`
6. **Age/duration calculation:** Use `TIMESTAMP_DIFF(end_ts, start_ts, DAY)` instead of `AGE()` or `EXTRACT(DAY FROM AGE(...))`.
7. **Date truncation:** Use `TIMESTAMP_TRUNC(ts, MONTH)` instead of `DATE_TRUNC('month', ts)`.
8. **Parameter placeholder comment:** `-- Parameterized: WHERE patient_id = @patient_id`

Apply these transformations to each of the 7 queries maintaining identical CTE structure and business logic.

**Specific file notes:**

- **vital_sign_trends.sql:** Same LOINC code filter. Use `PARSE_TIMESTAMP('%Y-%m-%dT%H:%M:%S%z', LTRIM(vital_sign_observation_time, ', '))` for timestamp.
- **lab_results.sql:** Use `SAFE_CAST(lab_value_quantity AS FLOAT64)` instead of `::NUMERIC`. Same commented reference ranges section.
- **care_team.sql:** Use `UNNEST(SPLIT(practitioner_role_id_references, ', ')) AS practitioner_role_id` instead of `unnest(string_to_array(...))`.
- **encounter_history.sql:** Use `TIMESTAMP_DIFF(end_ts, start_ts, DAY)` for duration instead of `AGE()`.
- **medication_timeline.sql:** Use `TIMESTAMP_DIFF` for duration_days instead of `EXTRACT(DAY FROM AGE(...))`.
