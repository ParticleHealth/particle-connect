2. Replace the `--source api` stub (lines 86-88) with working API integration:
   ```python
   if source == "api":
       if not patient_id:
           typer.echo(
               "--patient-id is required when --source api\n\n"
               "Usage: particle-pipeline load --source api --patient-id <patient-id> --target postgres"
           )
           raise typer.Exit(code=1)

       from observatory.api_client import ParticleAPIClient

       try:
           client = ParticleAPIClient()
       except ValueError as e:
           typer.echo(str(e))
           raise typer.Exit(code=1)

       try:
           raw_data = client.get_flat_data(patient_id)
       except Exception as e:
           typer.echo(f"API request failed: {e}")
           raise typer.Exit(code=1)

       # Apply same normalization as file mode for identical downstream behavior
       from observatory.normalizer import normalize_resource
       from observatory.parser import EXPECTED_RESOURCE_TYPES

       data = {}
       for key in EXPECTED_RESOURCE_TYPES:
           records = raw_data.get(key, [])
           data[key] = normalize_resource(records)
   else:
       # existing file loading code stays as-is
   ```

3. Move the existing file-loading block (`from observatory.parser import load_flat_data` through `data = load_flat_data(data_path)`) into the `else` branch of the `if source == "api"` conditional.

4. Update the command docstring to include an API source example:
   ```
   particle-pipeline load --source api --patient-id abc-123 --target postgres
   ```

5. The rest of the pipeline (inspect_schema, load into target, quality report) remains unchanged -- it receives `data` (dict[str, list[dict]]) from either source path.

**IMPORTANT:** Use deferred imports (inside the `if source == "api"` block) following the existing pattern in cli.py. This avoids importing api_client.py when running in file mode.
